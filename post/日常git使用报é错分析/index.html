[问题描述]

在使用Git Bash时出现的问题，使用 git add . 命令是将当前文件夹的所有文件放到暂存区，但有时会出现以下错误：

1
2

warning: LF will be replaced by CRLF in content/post/first.md.
The file will have its original line endings in your working directory.

[原因]

通过网上查找得知是因为Git字符转换时出错的，具体的字符网上说是换行符，也就是文件中的换行符存在差异而发出的警告，不影响使用。 Window默认的换行符是CRLF，而Git默认支持的是LF（Unix），所以在Window Commit时，git会把CRLF转换成LF， Update时转换成CRLF。

[疑问]

既然git会自动转换换行符格式，那为什么现在还是会报这个错误？还是没了解，需要再研究一下，反正先把问题解决了。

[解决方法]

1
2
3
4
5

git rm -r --cached .
git config core.autocrlf false
git add .
git commit -m ""
git push

答案来自CSDN网友博客
